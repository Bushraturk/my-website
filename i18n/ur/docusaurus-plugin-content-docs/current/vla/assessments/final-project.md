# فائنل پروجیکٹ: فزیکل AI اور ہیومنوائڈ روبوٹکس Capstone

## مقصد

کورس کے دوران سیکھے گئے تمام تصورات کو یکجا کرتے ہوئے ایک جامع Physical AI سسٹم ڈیزائن اور لاگو کریں جو embodied intelligence کا مظاہرہ کرے۔ اس capstone پروجیکٹ میں ROS 2، simulation environments، NVIDIA Isaac perception systems، اور Vision-Language-Action models کے علم کو ایک متحد روبوٹک ایپلیکیشن میں ملانا ضروری ہے۔

## سیکھنے کے نتائج

اس پروجیکٹ کے بعد آپ:
- چاروں کورس ماڈیولز کے علم کو ایک مربوط روبوٹک سسٹم میں ترکیب کر سکیں گے
- ایک پیچیدہ embodied AI ایپلیکیشن ڈیزائن اور لاگو کر سکیں گے
- متحد رویے کے لیے متعدد روبوٹک ذیلی نظام ضم کر سکیں گے
- اپنے سسٹم کی صلاحیتوں اور حدود کا جائزہ لے سکیں گے

## پروجیکٹ کی ضروریات

### سسٹم تصور (15 نمبر)
1. Physical AI اصولوں کا مظاہرہ کرنے والا مخصوص روبوٹک ٹاسک متعین کریں
2. یہ جواز دیں کہ آپ کا تصور کورس کے متعدد ماڈیولز کا انضمام کیسے ظاہر کرتا ہے
3. اپنے سسٹم کے لیے کامیابی کے metrics متعین کریں

مثال پروجیکٹ تصورات:
- خودکار گھریلو معاون جو قدرتی زبان کے احکامات سمجھ کر اشیاء کو manipulate کر سکے
- معائنہ روبوٹ جو پیچیدہ ماحول میں navigate کرے اور زبان کے ذریعے anomalies رپورٹ کرے
- collaborative manufacturing روبوٹ جو انسانی ہدایات کا real-time جواب دے
- تعلیمی روبوٹ جو interactive dialogue کے ذریعے روبوٹکس تصورات سکھائے

### تکنیکی Implementation (50 نمبر)

#### ROS 2 انٹیگریشن (10 نمبر)
- مناسب message passing کے ساتھ مضبوط ROS 2 آرکیٹیکچر
- مکمل سسٹم شروع کرنے کے لیے ROS 2 launch files
- مناسب error handling اور node lifecycle management

#### Simulation ماحول (10 نمبر)
- Gazebo/Unity simulation ماحول میں سسٹم کا مظاہرہ
- حقیقت پسندانہ physics modeling اور sensor simulation
- Physical testing سے پہلے simulation میں کام کی تصدیق

#### Perception سسٹم (10 نمبر)
- Scene understanding کے لیے NVIDIA Isaac perception pipeline
- متعدد sensor modalities کا انضمام
- Object detection، tracking، اور recognition کا مظاہرہ

#### Vision-Language-Action انٹیگریشن (10 نمبر)
- High-level command interpretation کے لیے VLA ماڈل
- زبان کی سمجھ کو perception اور action systems سے جوڑنا
- مبہم یا پیچیدہ احکامات کا سامنا

#### سسٹم انٹیگریشن (10 نمبر)
- تمام اجزاء کا متحد سسٹم میں مل کر کام کرنا
- انفرادی اجزاء کی ناکامی پر graceful degradation
- Safety measures اور emergency stopping

### دستاویزات (20 نمبر)
1. **سسٹم آرکیٹیکچر**: اجزاء کے تعامل کے ساتھ مکمل سسٹم diagram
2. **Implementation گائیڈ**: سسٹم reproduce کرنے کی مرحلہ وار ہدایات
3. **تکنیکی تفصیلات**: تفصیلی component descriptions
4. **کارکردگی کا تجزیہ**: متعین metrics کے خلاف سسٹم کارکردگی کا جائزہ

### جائزہ اور پریزنٹیشن (15 نمبر)
1. شماریاتی تجزیہ کے ساتھ مقداری نتائج
2. سسٹم صلاحیتوں اور حدود کی بحث
3. ممکنہ بہتری کا تفصیلی تجزیہ
4. سسٹم ڈیزائن اور implementation کی واضح وضاحت

## جمع کرانے کی چیزیں

### 1. Code Repository (40 نمبر)
- آپ کے integrated سسٹم کا مکمل source code
- سسٹم چلانے کے لیے تمام configuration files
- مکمل سسٹم شروع کرنے کے لیے launch files
- Setup اور usage ہدایات کے ساتھ README

### 2. Demonstration ویڈیو (25 نمبر)
- آپ کے سسٹم کی 5-10 منٹ کی ویڈیو
- سسٹم آرکیٹیکچر اور ڈیزائن choices کا جائزہ
- سسٹم آپریشن کا live demonstration
- چیلنجز اور حل کی بحث

### 3. تکنیکی رپورٹ (25 نمبر)
- آپ کے سسٹم کی وضاحت کرنے والی 5-8 صفحات کی رپورٹ
- سسٹم آرکیٹیکچر اور ڈیزائن کی وجوہات
- Implementation تفصیلات اور تکنیکی چیلنجز
- جائزہ کے نتائج اور کارکردگی کا تجزیہ

### 4. پریزنٹیشن سلائیڈز (10 نمبر)
- آپ کے پروجیکٹ کا خلاصہ کرنے والی 8-12 سلائیڈز
- واضح diagrams اور illustrations
- مقداری نتائج اور تجزیہ
