# دستاویزات کا ڈھانچہ

## دستاویزات کیوں اہم ہیں؟

دستاویزات کوائف کی طویل المدت کامیابی کے لیے اہم ہیں:

- کوائف کو سمجھنے میں مدد دیتی ہیں
- نئے شراکت داروں کو تیزی سے شامل ہونے کی اجازت دیتی ہیں
- مستقبل کی ترقی کے لیے رہنمائی فراہم کرتی ہیں
- تکنیکی مسائل کو حل کرنے میں مدد دیتی ہیں

## کوائف دستاویزات کا ڈھانچہ

ہمارے کوائف کو درج ذیل اقسام کی دستاویزات کی ضرورت ہے:

### 1. نصابی دستاویزات

- **کورس کا تعارف**: کورس کا سرخیل خلاصہ
- **ہر ماڈیول کے تعارفات**: ROS2، Gazebo، NVIDIA Isaac، VLA
- **ہر ہفتہ کی تفصیلات**: ہر ہفتے کا سیکھنے کا مواد
- **لیب مشقیں**: عملی کام کے لیے ہدایات
- **جائزے**: سیکھنے کی جانچ کے لیے

### 2. تکنیکی دستاویزات

- **کمپوننٹ ڈاکومنٹیشن**: ہر کمپوننٹ کی تفصیل
- **API ریفرنس**: کوائف کے API کی تفصیل
- **کنفیگریشن گائیڈز**: ترتیب اور استعمال کی ہدایات
- **ٹربل شوٹنگ**: مسائل کا حل

### 3. آپریشنل دستاویزات

- **ڈیپلائمنٹ رہنما**: کوائف کو ڈیپلائی کرنے کے لیے
- **مینٹیننس گائیڈ**: کوائف کو برقرار رکھنے کے لیے
- **سکیورٹی گائیڈ**: حفاظتی ترتیبات کے لیے
- **پرفارمنس مانیٹرنگ**: کارکردگی کی نگرانی کے لیے

## دستاویزات کے لیے بہترین طریقے

### 1. مسلسل اپ ڈیٹس

دستاویزات کو وقت کے ساتھ اپ ڈیٹ رکھیں:

- نئی خصوصیات کے لیے دستاویزات شامل کریں
- تبدیل شدہ خصوصیات کی دستاویزات کو اپ ڈیٹ کریں
- حذف شدہ خصوصیات کی دستاویزات کو ہٹا دیں

### 2. قابل رسائی دستاویزات

- سادہ زبان استعمال کریں
- مثالیں فراہم کریں
- تصاویر اور چارٹس شامل کریں
- تلاش کے قابل دستاویزات

### 3. جامع دستاویزات

ہر خصوصیت کے لیے:

- کیوں ضروری ہے؟
- کیسے استعمال کیا جائے؟
- کیا متبادل ہیں؟
- کیا مسائل ہو سکتے ہیں؟
- کیسے ٹربل شوٹ کیا جائے؟

## دستاویزات کا نظم و نسق

### Markdown سٹائل گائیڈ

ہم معیاری Docusaurus Markdown استعمال کرتے ہیں:

```markdown
---
title: Page Title
sidebar_position: 5
---

# Page Title

Content here...

## Section Title

More content...
```

### ترجمہ کا استعمال

```markdown
# <DOMTranslate>Original English Title</DOMTranslate>

<DOMTranslate>Original English content...</DOMTranslate>
```

## نتیجہ

دستاویزات کوائف کی کامیابی کا ایک اہم حصہ ہیں۔ جامع، جدید، اور قابل رسائی دستاویزات کے ذریعے، ہم یقینی بنا سکتے ہیں کہ:

- نئے صارفین تیزی سے شامل ہو سکتے ہیں
- موجودہ صارفین کوائف کو مؤثر طریقے سے استعمال کر سکتے ہیں
- تکنیکی مسائل کو جلدی حل کیا جا سکتا ہے
- کوائف کی ترقی کو جاری رکھا جا سکتا ہے
